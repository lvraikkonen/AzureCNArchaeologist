# Publish Historical SLA Versions as Version Resources

Status: Accepted

Historical SLA HTML pages are independently publishable version resources beneath one canonical Product Definition, with explicit version identities, language-specific Source Locations, CMS routes, payloads, and diagnostics. Their normalized inputs remain byte-identical to production snapshots while version-history links are rewritten only in Business Payloads from an explicit route table; separate Product Definitions and in-place normalized HTML edits were rejected because they would distort product counts or break source provenance.
