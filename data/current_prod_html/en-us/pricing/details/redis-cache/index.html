Azure Cache for Redis

Power applications with high-throughput, low-latency data access

Azure Cache for Redis is a dedicated service hosted by Azure which can generate highly scalable and highly responsive applications by providing ultra high-speed data access. You can also use Redis to offer abundant feature sets and ecosystem while obtaining reliable loading and supervision from Azure.

Premium

In addition to all standard level features, the new Premium tier includes even more other features, e.g.: performance superior to basic or standard tier caches, bigger workload, disaster recovery and enhanced security. Other features include:

  • Redis Persistence allows you to save data stored in the Redis Cache. You can also obtain screenshots and back up data, so as to upload them in case of failure.
  • Redis Cluster automatically segments data in multiple Redis nodes, thus you can create a workload (bigger than 53 GB) with bigger memory and obtain better performance.
  • Azure Virtual Network (VNet) is deployed to provide enhanced security for Azure Cache for Redis, and also offers subnet and access control policies and further limits other access features.

FAQ

Expand All
  • How are we charged for data transferred to and from the cache?

    Data transferred to and from the cache are charged at the normal rate .

  • Can I scale the Azure Cache for Redis Unit (similar to the concept of a Hosted Cache)?

    Yes. You can expand to multiple Redis Cache Units at the Advanced level.

  • Which kind of cache will be used if I want to create a new cache in Azure?

    Deploying Azure Cache for Redis is recommended due to its flexibility. This cache is based on an open standard Redis agreement supported by many programming languages (C/C++, .NET, Go, Java, Node.js, Objective C, Perl, PHP, Python, and Ruby, as well as over 29 more languages). Thus, almost any workload generated in Azure may use this cache.

  • Since you recommend using Redis, why the recommended “Hosted Caching” option is not provided?

    This aims to support the customers, who invest in the Shared Caching they rely on, giving them sufficient time to migrate to Redis Cache.

  • Is the storage billed separately?

    Yes. Premium Redis supports Redis Data Persistence, if customer set Data Persistence in Premium Redis, the storage will be charged separately.

    the bill of storage depends on the storage type customer selected. If he/she selected SSD, the storage would be charged based on the price of SSD; if he/she selected HDD, the storage would be charged bases on standard page blob.
