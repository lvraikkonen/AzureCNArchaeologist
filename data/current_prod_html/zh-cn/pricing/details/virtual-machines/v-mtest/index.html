IP 地址选项

每项包含一台或多台 Azure 虚拟机的云服务都被自动分配了一个免费的动态虚拟 IP (VIP) 地址。如果另外付费，你还可以获得：

  • 实例层级公共 IP 地址： 动态公共 IP (PIP) 地址，分配给虚拟机用于直接访问。
  • 保留的 IP 地址： 可以为订阅保留的公共 IP 地址，可以用作该区域内任何云服务的 VIP 地址。
  • 负载均衡的 IP 地址： 其他负载均衡的 VIP 地址，可以分配给包含一台或多台 Azure 虚拟机的云服务。

CPP

常规

  • 本地磁盘存储器是否单独计费？

    不会。所有新虚拟机都有操作系统磁盘和本地磁盘（也称“资源磁盘”）。我们不对本地磁盘存储器收费。操作系统磁盘按磁盘的常规费率收费。 请参阅所有虚拟机配置 。

  • 什么是临时磁盘？

    临时磁盘指直接附加到部署的 VM 的存储器。VM 关闭后，临时磁盘上的数据丢失。如果需要永久性存储，有各种类型和大小的永久型数据磁盘可供选择（这些磁盘单独收费）。请参阅 存储定价页面 了解相关信息。对于永久性 VM 存储，我们建议使用托管磁盘，以便获得更好的管理功能、可扩展性、可用性和安全性。

  • 如果我部署的实例显示“已停止”，还会计费吗？

    也许。如果状态显示为“已停止(已取消分配)”，则不会计费。如果状态显示为“已停止(已分配)”，则仍会针对已分配的虚拟核心（不是软件许可证本身）计费。可在此处查看虚拟机状态的所有详细信息。

    状态 已计费 详细信息
    正在启动 是 虚拟机的初始启动状态，此时它们正在经历启动周期。此期间将计费，因为虚拟机正在运行。
    正在运行（已启动） 是 虚拟机的正在运行状态。
    已停止 是 如果虚拟机已停止（但未取消分配），不对软件计费，但对内核计费。
    注：若要将虚拟机置于“已停止”状态，请使用虚拟机内部（而不是来自 Azure 经典门户）的电源选项。
    已停止（已取消分配） 否 不再对其进行收费。
    请注意 - 停止云服务中全部虚拟机的唯一方式是同时删除临时部署和生产部署。
  • 虚拟机实例价格是否包括与运行的镜像关联的存储成本？

    虚拟机价格仅包含本地磁盘存储。可在 托管磁盘 的定价页查看将高级磁盘（基于 SSD）和标准磁盘（基于 HDD）附加到虚拟机的费用。

  • A 系列实例如何计费？

    以前，A0、A2、A3 和 A4 标准和基本虚拟机的所有用量都发布为 A1 虚拟机计算分钟数的约数 (A0) 或倍数（A2、A3 和 A4）。我们了解到这给我们的客户造成了某些困扰，因此我们实行了更改，将每秒钟用量分配给专用的 A0、A2、A3 和 A4 计价器。转换的结果是，A0、A2、A3 和 A4 虚拟机的资源 GUID 将有所改变。如果你有任何计费例程依赖于资源 GUID，则你需要将其修改为将这些新的计价器纳入考虑。

  • 如果我的虚拟机运行了 6 分钟 45 秒，会向我收取多少费用？

    我们对虚拟机运行的整分钟数进行收费，因此你无需为任何额外的秒数付费。本示例中将按照 6 分钟的使用时间进行计费。

  • 什么是内部负载均衡？

    它是具有专用 IP 地址的 Azure 虚拟机的新预览功能。只有在虚拟网络（对于虚拟网络内的虚拟机）或云服务（对于虚拟网络外的虚拟机）内才能访问经过内部负载均衡的终结点。对于有些应用程序级别并不公开但仍要求负载均衡功能的多级应用程序，内部负载均衡是很有用的。

  • 如何检查是否已正确停止虚拟机并且将不对其计费？

    可以在“虚拟机”选项卡和虚拟机仪表板中检查虚拟机的状态。如果虚拟机状态显示为“已停止”，则仍会计费。仅“已停止(已取消分配)”表示不计费。

    要确保不被计费，请始终从管理门户停止虚拟机。还可以通过 Powershell 调用“PostShutdownAction”等于“StoppedDeallocated”的 ShutdownRoleOperation 来停止虚拟机。

  • 如果我在两个不同的区域都有实例，会如何向我收取数据传输费用？

    对每个实例跨区域的数据传出收费。

  • 上价格是否含税？

    是，所列价格均含税。

  • 能否将服务器许可证传输到 Azure 并在虚拟机上运行它们？

    可以，如果有软件保障，则可以通过许可移动性为所有虚拟机支持的服务器产品“自带许可”。

  • 何时使用 Azure SQL 数据库？何时使用 SQL Server？
