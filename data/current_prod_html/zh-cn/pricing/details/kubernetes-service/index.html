Azure Kubernetes 服务 (AKS)

作为完全托管的 Kubernetes 容器业务流程协调程序服务，Azure Kubernetes 服务 (AKS) 是一种免费的容器服务，可以简化 Kubernetes 的部署、管理和操作。仅针对所使用的虚拟机、相关存储和网络资源计费，使 AKS 成为市场上最有效、最具成本效益的容器服务。

添加运行时间服务级别协议的选项

受财务支持的服务级别协议 (SLA) 对使用 Azure 可用性区域的群集的 Kubernetes API 服务器保证 99.95% 的运行时间；对不使用 Azure 可用性区域的群集的 Kubernetes API 服务器保证 99.9% 的运行时间。

IP 地址选项

每项包含一台或多台 Azure 虚拟机的 Azure 云服务都被自动分配了一个免费的动态虚拟 IP (VIP) 地址。如果另外付费，你还可以获得：

  • 实例级别公共 IP 地址 - 动态公共 IP (PIP) 地址，分配给虚拟机用于直接访问。
  • 保留 IP 地址 - 可以保留并用作 VIP 地址的公共 IP 地址。
  • 负载均衡的 IP 地址 - 额外的负载均衡 VIP 地址，可以分配给包含一台或多台 Azure 虚拟机的 Azure 云服务。

常见问题

全部展开
  • 为什么我们启用了 AKS 之后，产生大量的 Azure 监控器费用？

    Azure 监视器提供了对容器及 AKS 节点非常强大和有用的日志记录和指标。也因此产生一定的数据提取成本。创建 AKS 时，请查看是否需要开启“监视器”如果不需要请手动禁用此功能，否则有可能会产生监视的费用，您可以参考 Azure 价格监控器价格详情 获得更多详细信息。

支持和服务级别协议

作为一项免费服务，AKS 不提供资金支持的服务级别协议。我们致力于让 Kubernetes API 服务器保持至少 99.5% 的可用性。虚拟机 SLA 涵盖群集中代理节点的可用性。有关详细信息，请参阅 虚拟机 SLA 。
