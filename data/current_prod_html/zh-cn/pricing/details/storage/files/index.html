Azure 文件存储提供无服务器文件共享，可通过 SMB、NFS 和 FileREST 协议进行访问。Azure 文件共享可由 Azure VM 中的客户端或运行 Windows、macOS 和 Linux 的本地工作站同时装载。此外，Azure 文件同步可用于缓存和同步 Windows 服务器上的 Azure 文件共享，以便本地访问。

Azure 文件存储支持两个不同的存储媒体层(SSD 和 HDD)，这让你能够根据自身场景的性能和价格要求定制文件共享:

  • SSD (“高级”): 对于大多数 IO 操作，固态硬盘(SSD)上托管的文件共享以个位数毫秒级提供一致的高性能和低延迟。
  • HDD (“标准”): 对于常规用途使用，硬盘驱动器(HDD)上托管的文件共享提供经济高效的存储。

Azure 文件存储提供多种定价模型，包括预配选项和即用即付选项。 以下部分详细介绍了每个模型的细节，以帮助你选择哪个模型最适合你的场景。

预配的 v2

在预配的v2模型中，Azure 文件的主要成本取决于您在创建或更新文件时预配的存储容量、IOPS 和吞吐量，而不取决于实际使用量。 尽管我们会根据您选择的预配存储量向您推荐 IOPS 和吞吐量，您可以任意分别预配存储、IOPS 和吞吐量。 我们建议所有新的 Azure 文件部署使用预配的v2模型。有关预配 v2 计费模型的更多信息，请参阅 预配 v2 计费模型。

预配的 v2 文件共享按小时计费，采用小时费率。

预配的 v1

在预配的v1模型中，Azure 文件的主要成本取决于您在创建或更新文件时预配的存储容量、IOPS 和吞吐量，而不取决于实际使用量。 在预配的v1 计费模型中，您只需为一个预配数量付费：预配存储。当您创建文件时，该文件可用的预配 IOPS 和吞吐量将根据您预配的存储容量确定。 如果您需要更多 IOPS 或吞吐量，则需要预配额外的存储。预配 v1 模型仅适用于 SSD 文件。我们建议新的 SSD 文件部署使用预配的v2 计费模型。 有关预配的v1 计费模型的更多信息，请参阅 “了解预配 v1 计费模型” 。

现用现付

在现用现付模式中，Azure 文件的成本取决于您使用文件的多少，包括已用存储、事务以及数据传输费用。 现用现付模式只支持 HDD 存储， 目前提供三种套餐，用于在已用存储与事务价格之间取得平衡：

  • 事务优化套餐：为事务密集型工作提供最低的事务成本，但不需要 SSD 文件所提供的低延迟。 推荐用于首次将数据迁移到现用现付模式的文件中。
  • 热访问套餐：为同时具有存储和事务需求的工作提供平衡的存储和事务价格。
  • 冷访问套餐：为存储密集型工作提供最具成本效益的存储价格。

现用即付仅适用于 HDD 文件共享。建议对新的 HDD 文件共享部署使用预配的 v2 计费模型。有关即用即付计费模型的详细信息，请参阅 了解即用即付计费模型。

  • 如果我创建一个带 5 TB 配额的共享，则向整个 5 TB 收费还是只收取所用存储空间的费用？

    对于标准文件共享，仅对所用存储空间计费，而非所设配额；高级文件共享的定价基于已预配的 GiB，其中包含所有事务成本。请注意：高级文件共享的预配大小按共享上的配额进行指定。如果将预配的共享大小减少到使用的 GiB 以下，将需要为高级共享上使用的 GiB 付费。
