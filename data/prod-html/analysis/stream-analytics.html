    流分析 Stream Analytics

    实时数据流式处理，可处理来自数百万台 IoT 设备的数据

    Azure 流分析 是云中的一种事件处理引擎，可实时地从设备、传感器、云基础结构和现有数据属性进行深入了解。通过与 事件中心 的集成，组合解决方案可以在对数据进行实时流式处理的同时接收数百万事件并进行分析，以更好地了解模式、增强仪表板或开始执行操作。

    定价详细信息

    Azure 流分析的价格取决于预配的流单元数。流单元代表分配给资源的内存量和计算量。你可以详细了解 所需的流单元数 。

    常见问题

    全部展开
    • 什么是流式处理单元？

      流式处理单元是流处理计算能力单位，最大吞吐量为 1MB/s。流式处理单元数由客户通过 Azure 门户或流分析管理 API 进行选择。流式处理单元可用于扩展流处理作业以便实现较高的吞吐量。根据查询的复杂性和所需吞吐量，可能需要更多流式处理单元才能实现性能要求。

      流式处理所用的单元数按小时计费，计费依据为在这一小时选择的最大单元数。

    • 影响流作业价格的因素有哪些？

      影响流作业价格的主要因素是查询复杂性、查询所需的返回速率（延迟）以及处理的数据量。

    • 可以选择的流式处理单元数是否有限制？

      流式处理单元的数量没有限制。每个区域单个订阅的默认流式处理单元配额为 500。您可以通过填写支持工单来请求更多数量的流式处理单元。

    • 运行流作业是否需要事件中心？

      建议将事件中心（Event Hub）作为可伸缩事件中转器，以用于流分析解决方案；不过流分析当前还支持从Azure Blob 拉入数据。我们将来可能会考虑与其他输入源集成。
