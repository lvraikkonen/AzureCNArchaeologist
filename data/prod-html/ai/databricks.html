Azure Databricks

基于 Apache Spark ™ 并针对 Azure 优化的快速、简单、协作分析平台

Azure Databricks 由 Microsoft 及 Apache Spark 的创建者合作设计，它结合了 Databricks 和 Azure 的优点，旨在通过针对 Azure 优化的高性能分析平台启用数据科学，以帮助客户加速创新。Azure Databricks 具有现成的 Azure Active Directory 集成、本机数据连接器、集成计费和符合性（即将推出）。Azure Databricks 还具有一键式设置、精简工作流和交互式工作区，因此，数据科学家、工程师和业务分析师可以相互协作。

工作负荷

Azure Databricks 在多个 VM 实例 上为通用计算工作流量身打造了 3 个不同的工作负载：借助作业计算和作业计算轻量工作负载，作业计算师可轻松生成和执行作业；借助通用计算工作负载，数据科学家可轻松以交互方式探索、直观显示、操作和共享数据及见解。

  • 什么是 DBU？

    Databricks 单位（“DBU”）是每小时处理能力的单位，按每秒使用量计费。

  • 作业计算和通用计算工作负载有何不同？

    作业计算工作负载指在其运行的群集上启动和终止群集的作业。例如，Databricks 作业计划程序可能会触发工作负载，该计划程序仅为该作业启动新的 Apache Spark 群集，并在作业完成后自动终止群集。

    通用计算工作负载指任何非自动化的工作负载，例如，在 Databricks 笔记本中运行命令。除非手动终止，否则这些命令将在 Apache Spark 群集上持续运行。多个用户可共享某个群集，以协作完成交互式分析。

  • 可否提供有关计费如何工作的相关示例？

    根据群集运行的工作负载类型，将收取作业计算或通用计算工作负载费用。例如，如果群集运行由 Databricks 作业计划程序触发的工作负载，则收取作业计算工作负载费用。如果群集运行临时命令等交互式功能，则收取通用计算工作负载费用。

  • 如果我的群集运行不到一小时，须支付多少费用？

    收费针对的是群集运行的分钟数（四舍五入到最接近的分钟），而不是小时。

  • Databricks 高并发池的工作原理是怎样的？

    仅 Azure Databricks 提供适用于通用计算工作负载的高并发池。标准层和高级层均提供高并发池。高并发池包含多个辅助角色节点和一个驱动程序节点。客户可选择任何可用 VM 实例作为其高并发池的辅助角色节点。此外，客户必须指定每个高并发池辅助角色节点的最小数量和最大数量。池中的所有辅助角色节点都必须是相同的 VM 实例。根据高并发池上的负载，Azure Databricks 会自动在指定的最小和最大辅助角色节点数内变换池中的实例数。将按池中每分钟运行的每个 VM 以及这些 VM 实例相应的 DBU 向客户收费。此外，该服务会为高并发池的驱动程序节点选择一个 VM 实例。将按 VM 和驱动程序节点相应的 DBU 费用向客户收费。
