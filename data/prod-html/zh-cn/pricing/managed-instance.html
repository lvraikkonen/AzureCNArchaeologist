    活动异地复制

    活动异地复制 是 SQL 数据库中的一项功能，可用于为主数据库创建连续同步的可读异地辅助数据库。主数据库和异地辅助数据库都需要具有相同的服务层级，而且强烈建议为异地辅助数据库配置与主数据库相同的备份存储冗余和计算大小(DTU 或 vCore)。每个异地辅助数据库的计费价格与具有相同配置的主数据库相同。始终对主数据库进行备份。如果为辅助数据库配置了不同的备份存储冗余，则在异地故障转移后，当异地辅助数据库升级为主数据库时，将根据在新的主数据库上选择的存储类型(RA-GRS、ZRS、LRS)对新备份进行存储和计费。

    每个异地辅助数据库可以是一个数据库，也可以是弹性池中的数据库。弹性池中的数据库位于单个服务器上，并以设定的价格共享一组资源，且在弹性池级别进行计费。客户可以将任意数量的辅助数据库添加到池中，最高不超过最大值，而无需产生额外费用。

    DTU

    • 基于 DTU 的模型会怎么样？

      基于 DTU 和 vCore 的模型将继续共存。我们应客户要求推出了基于 vCore 的模型，这些客户希望对数据库资源获得更大的透明度，并可分别缩放计算和存储。

    • 基于 DTU 的 SQL Database 存储价格是如何计费的？

      是按照客户预设的值，而不是实际的占用量。例如：客户设置的最大存储是 750GB，实际占用量是 418GB，P4 包含的存储量为 500GB，那么实际会收取 750-500=250 的费用。

    • 基于 DTU 的模型中的定价示例，用于预配了额外存储单个数据库

      假设 S3 数据库预配了 1 TB。S3 包含的存储量为 250 GB，因此额外的存储量为 1024 GB - 250 GB = 774 GB。标准层中的额外存储的单价约为 ¥1.0812/GB/月，因此额外的存储价格为 774 GB * ¥1.0812/GB/月 = ¥804.4128/月。因此，S3 数据库的总价为 eDTU ¥969.37/月 + 额外存储 ¥804.4128/月 = ¥1,773.7828/月。

    • 基于 DTU 的模型中的定价示例，用于预配了额外存储的弹性池

      假设 125 eDTU 高级池预配了 1 TB。125 eDTU 高级池包含的存储量为 250 GB，因此额外的存储量为 1024 GB - 250 GB = 774 GB。高级池中的额外存储的单价约为 ¥2.1624/GB/月，因此额外的存储价格为 774 GB * ¥2.1624/GB/月 = ¥1,608.8256/月。因此，该池的总价为池 eDTU ¥2,208.75/月 + 额外存储 ¥1,608.8256/月 = ¥3,817.5756/月。

    • DTU模式的数据库，自动备份是否是收费的？

      自动备份是免费的，相关信息请参考链接： https://docs.azure.cn/zh-cn/sql-database/sql-database-automated-backups

    • DTU模式下的长期保留备份是否有提供免费量？

      长期保留备份是不包含免费量的。

    VCore

    • 什么是 vCore？

      虚拟核心（vCore）表示服务器可用的逻辑 CPU，并提供两代硬件供客户选择。对于使用 Gen4 vCore 创建的服务器，vCore 基于 Intel E5-2673 v3（Haswell）2.4 GHz 处理器。

    • 基于 vCore 的模型有哪些优势？

      通过基于 vCore 的购买模型，可灵活、受控而直接地将本地工作负荷要求转换到云。允许根据工作负荷需求，独立缩放计算和存储资源。

      基于 vCore 的模型适用于 SQL 数据库托管实例、单一数据库和弹性池部署选项，且这些选项都可使用适用于 SQL Server 的 Azure 混合权益。因此，无论是否使用混合权益，基于 vCore 的模型内部的定价都适用：

      • 包含许可证：不包含面向 SQL Server 的 Azure 混合权益。此选项适用于不愿意将具有软件保障的现有 SQL Server 许可证应用于基于 vCore 的数据库部署选项的客户。
      • 基准费率：包含面向 SQL Server 的 Azure 混合权益的折扣价格。客户可以通过使用具有软件保障的 SQL Server 许可证来选择此价格。
    • 什么是虚拟化权益？

      虚拟化权益适用于基于 vCore 的 Azure SQL 数据库上的虚拟化工作负荷。使用它通过有效的软件保障转换 SQL Enterprise Edition 的 1 个许可证核心，以按照基准费率获得最多 4 个托管实例常规用途 vCore。这样，就可以采用极具成本效益的方式将虚拟化应用程序迁移到基于 vCore 的 SQL 数据库。例如，如果有 SQL Enterprise Edition 的 4 个核心许可证，就可以按基准费率获得最多 16 个常规用途 vCore。

    • SQL Database许可证是否收费？

      收费，且SQL数据库官网公示的价格已经包含了许可证的费用。

    • 能否在本地和云中使用 SQL Server 许可证？

      必须在本地或云中使用许可证，但有 180 天的宽限期，此期间可在本地和云同时使用许可证以促进迁移。

    • 什么是面向 SQL Server 的 Azure 混合权益？我有资格使用它吗？

      面向 SQL Server 的 Azure 混合权益可帮助你最大限度地提高当前许可证投资的价值，并加快向云迁移的速度。Azure 混合权益 SQL Server 是基于 Azure 的混合权益，可通过它使用具有软件保障的 SQL Server 许可证，从而支付更少的 SQL 数据库费用（“基准费率”）。通过适用于 SQL Server 的 Azure 混合权益，可使用所有基于 vCore 的选项：SQL 数据库托管实例、单一数据库和弹性池。即使 SKU 是活动的也可以应用此权益，但请注意，从你在门户中进行选择时起，就会应用基准费率。不会补发信用额度。

      通过在 Azure 门户证明你有足够的具有软件保障的有效许可证，即可激活 Azure 混合权益。Azure 混合权益的工作原理如下：

      • 如果拥有含软件保障的、按核心许可证颁发的 Standard Edition，选择常规用途选项，每有 1 个本地许可证核心可获得 1 个 vCore。
      • 如果拥有含软件保障的、按核心许可证颁发的 Enterprise Edition，选择业务关键选项（即将推出），每有 1 个本地许可证核心可获得 1 个 vCore。注意：适用于 SQL Server 的 Azure 混合权益，其业务关键选项仅对 Enterprise Edition 客户提供。
      • 如果拥有含软件保障的、按核心许可证颁发的高度虚拟化 Enterprise Edition，选择常规用途选项，每有 1 个本地许可证核心可获得 4 个 vCore。这是仅限于 Azure 的独特虚拟化权益。
    • 能否以追溯方式应用面向 SQL Server 的 Azure 混合权益？

      否。Azure 混合权益不能以追溯方式进行应用。

    • 如果软件保障到期，面向 SQL Server 的 Azure 混合权益会发生什么情况？

      若要使用面向 SQL Server 的 Azure 混合权益，需要有效的软件保障。如果软件保障到期并且你没有续订，则系统会将你转到相应 SKU 的“包含许可证”定价。

    • 帐单中如何显示使用情况？

      在基于 vCore 的模型中，服务根据中 vCore 预配的计算、预配的存储（GB/月）及使用的备份存储，按可预测的每小时费率计费。如果备份超过数据库总大小（即数据库大小的 100%），将产生额外费用。 vCore 小时数、已配置数据库存储、已使用 IO 和备份存储在帐单中逐项列出，让你可以更轻松地看到所用资源的详细信息。包含最大数据库大小的至多 100% 的备份存储，超过后按每月使用的 GB 数计费。

      例如：

      • 如果 SQL 数据库在一个月内存在 12 小时，帐单将显示使用 vCore 12 小时的费用。如果 SQL 数据库预配了额外的 100 GB，则帐单会以 GB/月为单位显示按小时分摊的存储使用情况及一个月内使用的 IO 数。

      如果 SQL 数据库运行时间少于 1 小时，则根据该小时内所用的最高服务层和预配存储和 IO 来计算数据库每小时运行的费用，而不考虑使用情况或数据库运行时间是否少于 1 小时。

      例如：

      • 如果创建托管实例并在 5 分钟后删除，则按 1 数据库小时收费。
      • 如果在常规用途层创建一个具有 8 个 vCore 的托管实例，并立即将其升级为 16 个 vCore，则第一个小时按 16 个 vCore 收费。

      请注意：在 2018 年 6 月 30 日之前的有限期内，将不收取备份费用和 IO 费用。

    • 无服务计算层与 SQL 数据库提供的计算层有何不同?

      无服务层和已供应的计算层之间的关键区别在于计算资源的供应和计费。在无服务计算层中，计算资源根据工作负载的需求动态伸缩，并按每秒计费，在不活动期间暂停。在提供的计算层中，客户购买一定数量的计算资源，并按小时计费。

      无服务计算层通常更适合单个数据库，这些数据库的使用模式各不相同，中间穿插着一些不活动时期，或者数据库的计算资源使用情况难以预测。这些可以从基于计算资源使用量的每秒计费所带来的节省中受益。

      提供的计算层通常更适合:

      • 随着时间的推移，具有更统一和更可靠的计算利用率的单个数据库。在提供的计算层中，使用这种使用模式的数据库通常比较便宜。
      • 具有变化的和不可预测的计算使用的多个数据库，可以合并到一个弹性池中以实现更好的价格优化。
    • 无服务支持哪些购买模型和服务层?

      无服务器是基于 vcore 的购买模型和单个数据库服务层提供的。

    • 无服务下的是如何计费的?

      总账单是计算账单和存储账单的总和

      计算的计费是基于计算资源的实际使用情况按每秒计费的(有关详细信息，请参阅下一个问题:“ 如何在无服务计算层中计算账单? ”)。

      对于无服务器层和已供应的计算层，对给定服务层中的存储进行计费是相同的。

    • 如何在无服务计算层中计算账单?

      计算根据实际使用情况按每秒计费，并根据数据库联机时提供的最低计算资源计费。

    综合

    • 托管实例如何定价？

      托管实例允许独立缩放计算和存储。客户需要对计算（以 vCore 为单位）、存储空间和备份（以 GB 为单位）以及使用的 IO 数量付费。

      托管实例有两个价格选项：

      • 许可证已包含：面向 SQL Server 的 Azure 混合权益除外。此选项适用于不愿意将具有软件保障的现有 SQL Server 许可证应用于托管实例的客户。
      • 基本费率：包含面向 SQL Server 的 Azure 混合权益的折扣价格。客户可以通过使用具有软件保障的 SQL Server 许可证来选择此价格。
    • 数据库如何停止计费？

      删除数据库后计费停止。

    • 是否需要承担任何网络数据传输费用？

      公共预览期间网络数据入口或出口不收取任何附加费用。正式发布后将收取附加费用。

    • 必须移动到基于 vCore 的模型吗？

      不需要移动到基于 vCore 的模型。弹性池和单一数据库的基于 vCore 的部署选项简介体现了我们对客户选择和灵活性的承诺。如果希望继续使用基于 DTU 的模型，不需要采取任何操作，你的体验和计费方式都会保持不变。除基于 vCore 的模型外，基于 DTU 的模型仍是购买 SQL 数据库的可行选择。

    • 如何选择基于 DTU 或 vCore 的模型？

      数据传输单元（DTU）基于 CPU、内存、读取和写入的混合衡量。基于 DTU 的性能级别代表驱动不同级别应用程序性能的预配资源捆绑。如果不想担心底层资源，而希望使用简单的预配捆绑而每月支付固定费用，基于 DTU 的模型可能更能满足你的需求。

    • 是否需要将应用程序脱机，才能将基于 DTU 的数据库转换为基于 vCore 的服务层？

      新的服务层是简单的在线对话方式，与现有标准和高级服务层间的升级和降级过程类似。可使用门户、ARM、PowerShell、Azure CLI 或 T-SQL 启动门户。
