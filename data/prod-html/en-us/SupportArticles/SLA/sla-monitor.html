The SLA details

Monthly Uptime Calculation and Service Levels for the Azure Monitor managed service for Prometheus

Additional Definitions:

"Maximum Available Minutes" is the total number of minutes that a given Azure Monitor workspace has been deployed by Customer in a given Microsoft Azure subscription during a billing month.

"Downtime" is the total number of minutes within Maximum Available Minutes that data in a given Azure Monitor workspace are unavailable. A minute is considered unavailable for a given Azure Monitor workspace if all continuous attempts to retrieve Prometheus metric data throughout the minute result in either an Error Code or do not return any HTTP code within 60 seconds.

"Monthly Uptime Percentage" for a given Azure Monitor workspace is calculated as Maximum Available Minutes less Downtime divided by Maximum Available Minutes multiplied by 100. Monthly Query Availability Percentage is represented by the following formula: 

Maximum Available Minutes – Downtime / Maximum Available Minutes X 100

Service Levels and Service Credits:

Monthly Uptime Percentage Service Credit
< 99.9% 10%
< 99% 25%
