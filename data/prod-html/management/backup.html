    综合概述

    • 定价模型如何工作？

      Azure 备份的新定价模型有两个组成部分：

      • 受保护实例：这是 Azure 备份的主要计费单位。客户为使用 Azure 备份服务保护的实例的数量付费。
      • 存储器：客户可以选择 本地冗余存储 (LRS) 或异地冗余存储 (GRS) 作为他们的备份保管库。存储器的净价取决于使用服务存储的数据量。

      这两个组成部分作为单独的行项出现在 Azure 月度帐单上。

    • 受保护实例是什么？

      受保护的实例表示用于配置到 Azure 的备份的计算机、物理/虚拟服务器或者数据库。为计算机、服务器或数据库设置备份策略，并为数据创建副本后，实例即受到保护。备份数据的后续副本将更改使用的存储量，但额外备份副本不会使受保护的实例数量增加。受保护的实例的一些常见示例包括运行 Windows 操作系统的虚拟机、应用程序服务器、数据库和个人计算机。例如：

      • 运行 Hyper-V 或 Azure IaaS 虚拟机监控程序结构的虚拟机。这些虚拟机的来宾操作系统可以是 Windows Server 或 Linux。
      • 包含运行 Windows Server 的物理计算机或虚拟机的应用程序服务器，以及包含要备份的数据的工作负荷。常见工作负荷有 Microsoft SQL Server、Microsoft Exchange 服务器、Microsoft SharePoint 服务器、Microsoft Dynamics 和 Windows Server 的文件服务器角色。要备份或保护这些工作负荷，需要 Azure 备份服务器或 System Center Data Protection Manager (DPM)。
      • 运行 Windows 操作系统的个人计算机。
      • SQL 服务器，包括服务器上的所有数据库。

      如果选择备份虚拟机监控程序结构及其来宾工作负荷，则虚拟机监控程序结构和其来宾工作负荷将被视为单独的受保护实例并独立计费。

    • 是否所有受保护实例的定价都一样？

      不是，从计费角度来看，基于实例的大小将实例分类为三个类别。每个受保护实例的价格取决于以上定价表中实例的分类。请注意，上表中的价格不包括存储器的价格，存储器单独收费。

    • 我需为还原和网络出口付费吗？

      将不会针对任何还原操作或与还原操作关联的出站网络带宽（出口）向客户收费。

    • 哪些客户将获得此新定价？

      新定价将适用于所有 Azure 备份客户。

    受保护的服务器和实例

    • 如果我从月中开始在 Azure 上保护计算机，我仍然需要按受保护实例的全价付款吗？

      不需要，根据该实例受保护当月的使用天数按比例分摊费用。

    • 如果我增加我的备份数据的保留期并使用 Azure 备份存储更多数据，会发生什么？

      使用 Azure 备份存储的所有数据均按上述备份存储费率，对使用 Azure 备份保存数据的持续时间进行计费。但是，增加的保留期不影响你付费的受保护实例的软件价格。请注意，你必须在计算机上安装最新的 Azure 备份代理才能确保此行为。

    • 如果停止保护但将数据保留在 Azure 备份保管库中会怎么样？

      因为只要通过 Azure 备份保留了数据，你将继续为受保护实例和存储器付费。受保护实例账单基于你决定停止保护之前上次成功备份的大小。存储器账单基于该实例存储在备份保管库中的数据总量。

      例如，对于 100GB 虚拟机的备份。受保护实例收费为每月 ￥63.29。如果每天添加 1GB 的数据且数据保留 120 天，你将为此部分存储器单独付费。一个月后，如果你决定停止保护虚拟机但保留 Azure 备份中的数据，则每月将向你收费 ￥63.29，直到你将备份数据显式删除。

    • 我是否需要升级 Azure 备份代理？

      需要，Azure 备份代理需要升级到最新版本才能使用新定价。使用较早版本的代理将对要计费的受保护实例的大小产生不利影响。

    • 哪些客户将获得此新定价？

      新定价将适用于所有 Azure 备份客户。

    备份存储成本

    • 采用了高级存储的虚拟机备份有何提示？

      如果备份的虚拟机实例使用的磁盘类型为高级存储，那么在启动备份功能时，备份服务在高级存储帐户中会创建名为“AzureBackup-”的临时暂存位置。 暂存位置大小与恢复点快照大小相同。暂存的存储价格与高级存储价格一致。详情请参见： 结合使用高级存储 VM 和 Azure 备份 。

    • 我将在存储侧为哪些项付费?

      使用 Azure 备份存储的所有数据均按上述备份存储费率进行计费。

      由于启用Azure备份即时还原功能而产生的增量快照存储费用按照 定价页 收费。

    • 块 Blob 定价是否适用于 LRS 或 GRS 存储？

      这取决于为备份保管库选择的存储选项。备份保管库的存储选项有：本地冗余存储 (LRS) 或异地冗余存储 (GRS)。

    • 我能否拥有两个不同的保管库 - 一个使用 LRS 另一个使用 GRS？

      可以。每个保管库中的备份数据将根据保管库所在区域的相应 LRS 或 GRS 块 Blob 费率计费。客户对选取了保管库选项的 Azure 存储器帐单具有完全控制。

    • 什么有助于使用的备份存储？

      为特定受保护实例保存在 Azure 备份中的总备份数据量（“后端大小”）根据以下因素变动：

      • 备份数据的保留期
      • 存储在每个恢复点中的网络新流失
      • 备份数据可实现的压缩级别
      • 计划备份作业的频率
      • 任意相关联的备份元数据
    • 我如何查明使用的备份存储量？

      备份保管库仪表板页面将显示注册到保管库的计算机中的总数据量。它还会显示用于备份保管库的存储类型。

    • 我是否必须为存储事务付费？

      与当前定价一样，你无需为 Azure 备份的存储事务付费。这些内容作为你所支付软件价格的一部分包含在内。

    网络成本

    • 我已经注册了 ExpressRoute。我必须付出什么？

      ExpressRoute 费用不包含在 Azure 备份中。客户将继续单独为 Azure 备份付费，并根据选择的端口速度为 ExpressRoute 付费。
